# 🐍 [2026-01] 나만의 파이썬 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
*  지역별 대기오염을 진단하는 지표 시스템
<br>


## 2. 시나리오 (5~10줄)

 이 프로그램은 시민들의 호흡기 건강을 지키기 위해 전국 지역별 지표에 근거한 수치를 기반으로 대기오염을 진단하고 관리하는 시스템입니다.

 가장 먼저 관제 시스템에 측정하고자 하는 지역의 이름을 입력하고, 그곳의 미세먼지, 초미세먼지, 오존 농도등 오염 수치가 어느 정도인지 물어보고 그 대답을 기록하는 것으로 진단이 시작됩니다.
그 후로 입력된 수치들에 인체 유해도를 반영한 가중치를 곱하고 모두 합쳐서, 이 지역의 진짜 대기 상태가 어느 정도인지 '종합 오염 지수'를 계산해 봅니다.
이렇게 계산된 오염 지수가 낮으면 숨쉬기 좋은 1등급(최상) 청정구역으로 매겨주고, 점수가 높으면 당장 외출을 자제해야 하는 5등급(위험) 지역으로 판정해 줍니다.
만약 전반적인 수치는 낮더라도 인체에 치명적인 초미세먼지나 오존 수치 중 하나가 기준치를 뚫고 올라가는 특이한 지역이 있다면, '호흡기 파괴구역'이라는 특별 경보와 함께 마스크 착용 권고 같은 맞춤형 가이드를 제시하여 경각심을 일깨울 수 있도록 할 것입니다.
나중에는 단 하나의 지역만 기록하는 게 아니라, 우리 시스템을 거쳐 간 수많은 지역의 측정 데이터를 커다란 데이터베이스(리스트)에 한꺼번에 담아서 관리하고, 언제든 전국 대기질 명부를 꺼내 비교해 볼 수 있게 만들 것입니다.

 마지막으로 이 모든 기능은 관리자가 메뉴판에서 번호를 눌러 자유롭게 선택할 수 있고, 시스템 종료를 선택하기 전까지는 계속해서 새로운 지역의 데이터를 입력받고 진단할 수 있는 대기 환경 관제 소프트웨어로 완성할 계획입니다.

<br>


## 3. 예상 기능 및 메뉴 (최소 5개)

1. 신규 지역 대기질 데이터 입력 
2. 가중치 기반 종합 오염 지수 산출 
3. 5단계 대기질 위험 등급 판정 
4. 특수 경보 및 행동 요령 발령
5. 전국 대기질 통합 데이터 조회 
6. 최고/최악의 공기질 지역 분석 
7. 대기오염 관제 시스템 종료

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**

이번 과제는 지역별 대기오염 진단 시스템 구현으로, 사용자로부터 온도와 다수의 대기오염 데이터를 입력받아 리스트에 저장하고, 산술 연산과 내장 함수를 통해 통계값을 도출하는 시스템을 구축하는 것입니다. 특히 반복문과 제어문(break, continue)을 결합해 데이터를 필터링하고, if-elif 조건문과 and 논리 연산자를 활용하여 상황에 맞는 대기 상태 등급과 긴급 경보를 발령하는 지능형 로직을 완성해야했습니다.
input()과 for 반복문을 사용하여 관리자 정보, 지역, 온도 및 3종의 대기오염 농도를 입력받고 append()를 통해 리스트에 저장.
조건문 내에 break와 continue를 사용하여 음수 입력 시 데이터를 스킵하고, 999 입력 시 수집을 조기 종료하는 예외 처리 및 필터링 기능 적용.
리스트 내장 함수(sum(), len(), max())를 적용하여 수집된 데이터의 총계, 평균, 최댓값을 산출.
if-elif-else를 사용하여 산출된 평균값을 기준으로 '좋음'부터 '매우 나쁨'까지 4단계 대기 상태 판정 로직 추가.
and 논리 연산자와 중첩 if문을 활용하여 평균 및 최고치 기준을 동시에 초과할 경우 '[경고]' 및 '[긴급(야외 활동 금지)]' 알림을 출력하는 심화 경보 기능 구현.

## 🟨 [3차 과제: V3.0] 무한 루프와 메뉴 시스템 (반복문) - 향후 작성 예정
### **✨3차 과제 업데이트 내용:**
이번 3차 과제는 기존에 구축한 '지역별 대기오염 진단 시스템'을 기반으로, 함수(def)를 활용한 모듈화 및 무한 루프 기반의 사용자 친화적 메뉴 시스템을 도입하여 프로그램을 구조화하는 것입니다. 특히 전역 변수와 지역 변수의 위를 엄격히 분리하고, 매개변수와 반환값을 통해 데이터의 흐름을 체계적으로 제어하는 심화 프로그래밍 로직을 완성해야 했습니다.
while True 무한 루프를 사용하여 데이터 입력, 분석, 조회, 프로그램 종료를 선택할 수 있는 대화형 메인 메뉴를 구축하였으며, 사용자가 지정된 '종료' 번호를 선택했을 때만 break문이 작동하여 안전하게 루프를 탈출하도록 예외 없는 종료 로직을 구현.
변수의 생명주기를 고려하여 함수 내부에서만 사용되는 임시 데이터는 '지역 변수'로 선언해 메모리 효율을 높임. 반면, 프로그램 전반에서 공유되어야 할 핵심 데이터는 '전역 변수'로 관리하되, 입력 함수 내에서 global 키워드를 선언하여 외부 변수의 값을 안전하게 갱신하도록 변수의 역할을 명확히 구분 및 통제함.
기존 절차지향적으로 나열되었던 코드(입력, 연산, 출력 등)를 input_data(), analyze_data(), print_result() 등 3개 이상의 독립된 함수(def)로 분리하여 코드의 가독성과 재사용성을 극대화함. 이를 통해 while문 내부의 메인 로직은 마치 책의 목차처럼 함수 호출 위주로 간결하게 재구성함.
함수 간의 유기적인 데이터 교환을 위해, 연산 함수 정의 시 데이터를 매개변수(인자)로 전달받아 처리하도록 설계(analyze_data(data_list))하고, 산출된 분석 결과(평균, 최댓값)는 return 문을 사용하여 튜플 형태로 메인 로직에 반환하도록 구현.
